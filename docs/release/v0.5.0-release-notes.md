# Voyalier v0.5.0 — Traveler-owned planning

Voyalier 0.5.0 fills the practical space between a confirmed reservation and a
half-made idea without blurring the evidence boundary. Saved places, packing
items, and manual plans belong to the traveler; confirmed facts remain the only
inputs that can drive logistics readiness.

## Highlights

- Save persona weights and shortlisted recommendations with source, license,
  reasons, coordinates, and private notes. Removing the source pack does not
  remove the traveler's saved place.
- Turn evidence-backed packing suggestions into checklist items only after an
  explicit click, alongside custom items and persistent checked state.
- Add, edit, and remove activities, rail legs, and transfers. Scheduled items
  can appear in Today, the printable brief, and `.ics` export; private notes do
  not enter either export model. Planning overlaps are notices and never change
  confirmed-plan readiness.
- Search documents, confirmed facts, trip notes, saved places, and manual plans
  across the local workspace. Results always state their trip and source kind;
  pending extraction candidates remain excluded.
- Inspect the shipped data-source, license, attribution, and authority register
  from Settings.
- Switch immediately among System, English, and Español. The preference stays
  on this device; the Spanish catalog covers the primary application shell and
  v0.5 workflows, with safe English fallback for secondary copy.
- Use the expanded bounded offline-map path for Nashville, Kyoto, Tokyo, and
  Paris. The separate `packs-v1` pre-release remains the download surface for
  map and city-pack assets.

## Trust boundaries kept intact

- A saved suggestion or manual item is not a booking.
- Packing suggestions never add themselves.
- Planning overlap notices do not clear or worsen confirmed-plan readiness.
- Workspace search is local and vault-aware; no query or index is uploaded.
- Manual-item private notes have no field in the brief or calendar projection.
- Advice, weather, maps, packs, AI, and update checks retain their existing
  explicit network actions.

## Install

Download the Apple Silicon macOS `.dmg` or Windows x64 `.exe`/`.msi` from the
[GitHub release](https://github.com/udhawan97/Voyalier/releases/tag/v0.5.0).
The installers are not yet OS code-signed, so follow the documented
[first-launch steps](https://udhawan97.github.io/Voyalier/download/). Updater
artifacts are independently minisign-signed and verified by Voyalier.

## Verification added in this release

The existing Rust, migration, vault, live Axum serialization, Tauri/HTTP route
parity, React, accessibility, and build gates now include a Playwright Chromium
journey through the real loopback development stack. The offline-map publisher
also has pure Node coverage proving that every enabled map is built into one
manifest without descriptor loss.

See the [CHANGELOG](../../CHANGELOG.md) for the complete release history and
tradeoffs.
