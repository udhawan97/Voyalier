# Voyalier v0.5.1 — Calm-flow reliability

Voyalier 0.5.1 is a focused reliability release for the trip workspace. It does
not add a new planning authority, provider, or data format; it makes the existing
review, recovery, navigation, appearance, and planning flows behave as one
coherent interface.

## Fixed

- Evidence review opens at the title and top of the dialog instead of scrolling
  to a deep **Confirm** button. Resolving suggestions still advances through the
  queue, and completing it returns keyboard focus to the Blueprint.
- Engine reachability is consistent across the topbar and work area. A transport
  failure reports **Offline**, and a successful retry returns the shell to
  **Ready** without relabeling validation or storage errors as outages.
- The topbar uses a complete two-row mobile layout at 320–640 px, keeping search,
  health, theme, and Settings visible with 44 px targets and no horizontal page
  scroll.
- Topbar and Settings theme controls now share one Light/System/Dark preference,
  update in the same render, and retain the existing device-local persistence.
- Blank planning submissions are no longer silent: custom packing waits for
  text, and a manual plan names and focuses its required field.
- Reloading returns to the active trip and selected long-page section. Only the
  trip id is kept in session storage; trip content, evidence, notes, and searches
  remain in the normal local workspace.

## Trust boundaries kept intact

- Confirmation candidates still require a traveler decision before becoming
  facts.
- A checklist item or manual plan is still traveler-authored, never inferred as
  a booking.
- The re-entry hint is session-only navigation state, not a second data store.
- No contract, route, migration, provider, telemetry, or remote-service behavior
  changed.

## Install

Download the Apple Silicon macOS `.dmg` or Windows x64 `.exe`/`.msi` from the
[GitHub release](https://github.com/udhawan97/Voyalier/releases/tag/v0.5.1).
The installers are not yet OS code-signed, so follow the documented
[first-launch steps](https://udhawan97.github.io/Voyalier/download/). Updater
artifacts are independently minisign-signed and verified by Voyalier.

## Verification in this release

The React suite covers focus lifecycle, global transport recovery, shared theme
state, planning prerequisites, and session re-entry. The real loopback Playwright
journey additionally checks a 320 px topbar, synchronized duplicate theme
controls, section-aware reload, and persisted planning through Axum and SQLite.
The full Rust, desktop, docs, integration, security, packaging, updater-artifact,
checksum, provenance, and live-download gates remain release requirements.

See the [CHANGELOG](../../CHANGELOG.md) for the complete release history and
retained limitations.
