title: Roadmap
description: The sequence from foundation to public beta.

## Foundation — done

Repository, architecture, contracts, design system, CI, security, docs, web shell, local API, and desktop shell.

## Phase 1 — first vertical slice — done

Create trip → deterministic Blueprint → add/import reservation → readiness action → redacted brief.

## Phase 2 — grounded intelligence — done

Consent-gated official advice (FCDO) and weather (Open-Meteo) with freshness labels; deterministic trip search; offline city packs (Overture + Wikivoyage, per-layer licenses, CI-built, downloaded per trip with consent); and BYOK AI assist — on-device Ollama plus OpenAI/Anthropic — with a redaction preview, an activity log, and keys kept only in the OS keychain.

## Phase 3 — public beta — in progress

Shipped: an encrypted vault (keychain default, optional passphrase) with portable encrypted backup/restore; consent-gated maps; an offline Today view; persisted persona weights and saved places; an explicit packing checklist; manual activities, rail legs, and transfers kept separate from confirmed evidence; workspace-wide local search; a product-visible source/license register; English and Spanish locale preferences; confirmation import from paste, HTML, email/`.eml`, or a picked/dragged file; guided Ollama and validated BYOK cloud AI setup; editable AI instructions and reviewable lodging-date drafts; self-hosted fonts; the minisign-verified in-app updater with checksums/provenance; and verified offline PMTiles targets for Nashville, Kyoto, Tokyo, and Paris. Remaining: OS-signed and notarized desktop installers (DMG/MSI), blocked on paid code-signing certificates and independent of the updater above; more locales; and incremental expansion of bounded offline-map coverage.

## Later

Licensed live inventory, encrypted synchronization, group collaboration, monitoring, and mobile experiences.
